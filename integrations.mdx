## Overview

Extend your Yogesh Agrawal documentation with popular third-party services. Integrate with GitHub for version control, Slack for real-time notifications, and use webhooks or the API for custom workflows. You can also export docs to formats like PDF or Markdown.

## GitHub Integration

Connect your Yogesh Agrawal space to GitHub to import repositories or push changes directly.

<Tabs>
  <Tab title="Import Repo" icon="download">
    <Steps>
      <Step title="Authorize GitHub">
        Go to Settings > Integrations and select GitHub. Grant access to your repositories.
      </Step>
      <Step title="Select Repository">
        Choose the repo containing your Markdown docs.
      </Step>
      <Step title="Sync Pages">
        Map files to doc pages and trigger initial sync.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Push Changes" icon="upload">
    <CodeGroup tabs="CLI,API">
    ````bash
    yogesh-docs push --repo owner/repo --branch main --path docs/
    ````
    ````javascript
    const YogeshSDK = require('yogesh-sdk');
    const sdk = new YogeshSDK({ apiKey: 'YOUR_API_KEY' });
    await sdk.integrations.github.push({
      repo: 'owner/repo',
      branch: 'main',
      path: 'docs/'
    });
    ````
    </CodeGroup>
  </Tab>
</Tabs>

## Slack Integration

Receive notifications in Slack for doc events like publishes or comments.

<Callout kind="tip" title="Quick Setup">
  Use the built-in Slack app or webhooks for custom channels.
</Callout>

1. Install the Yogesh Agrawal Slack app from the Slack App Directory.
2. Add it to your workspace and select channels.
3. Configure events: page updates, new comments, user mentions.

For advanced setups, use incoming webhooks:

````javascript
// Example Slack webhook payload
{
  "text": "New doc published: *Introduction* by @user",
  "channel": "#docs",
  "icon_emoji": ":book:"
}
````

## Setting Up Webhooks

Webhooks notify external services of events in your Yogesh Agrawal space.

### Create a Webhook

<Steps>
  <Step title="Navigate to Webhooks">
    Go to Settings > Webhooks > New Webhook.
  </Step>
  <Step title="Configure Endpoint">
    Enter your URL, e.g., `https://your-webhook-url.com/yogesh`.
  </Step>
  <Step title="Select Events">
    Choose triggers like `page.updated` or `comment.created`.
  </Step>
  <Step title="Test and Save">
    Send a test payload and activate.
  </Step>
</Steps>

<Request tabs="cURL,JavaScript" show-lines="true">
````bash
curl -X POST https://api.example.com/v1/webhooks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-webhook-url.com/yogesh",
    "events": ["page.updated", "comment.created"]
  }'
````

````javascript
const response = await fetch('https://api.example.com/v1/webhooks', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    url: 'https://your-webhook-url.com/yogesh',
    events: ['page.updated', 'comment.created']
  })
});
````
</Request>

<Expandable title="Sample Webhook Payload" default-open="false">
```json
{
  "event": "page.updated",
  "spaceId": "space_123",
  "pageId": "page_456",
  "userId": "user_789",
  "timestamp": "2024-01-15T10:30:00Z"
}
```
</Expandable>

## Exporting Documentation

Export your docs to PDF, HTML, or Markdown.

<Tabs>
  <Tab title="Bulk Export" icon="download">
    From the dashboard, select Export > Format > Download.
  </Tab>
  <Tab title="API Export" icon="code">
    ````bash
    curl "https://api.example.com/v1/export/pdf?spaceId=YOUR_SPACE_ID" \
      -H "Authorization: Bearer YOUR_API_KEY" \
      --output docs.pdf
    ````
  </Tab>
</Tabs>

<Callout kind="info">
  Exports include all published pages. Private pages require additional permissions.
</Callout>
